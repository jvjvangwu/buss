# 业务钩子函数使用规则

## 1. 钩子函数选择原则

### 1.1 业务场景匹配
- **根据功能需求选择钩子**: 明确业务需求的核心功能，选择最适合的钩子函数
- **考虑代码复用**: 优先使用能够提高代码复用性的钩子函数
- **遵循项目规范**: 使用符合项目代码风格和命名规范的钩子函数

### 1.2 钩子函数优先级
1. **核心钩子**: 优先使用 `src/hooks/core/` 下的钩子函数
2. **自定义钩子**: 其次考虑根据业务需求定制钩子函数

## 2. 钩子函数与业务场景匹配

### 2.1 认证相关场景

#### 用户认证与权限管理
- **推荐钩子**:
  - `useAuth`: 处理用户认证、权限管理
- **使用建议**:
  - 在需要权限控制的页面组件中使用
  - 利用其提供的权限检查方法验证用户操作权限
  - 结合路由守卫使用，实现页面级权限控制

### 2.2 数据展示场景

#### 表格数据处理
- **推荐钩子**:
  - `useTable`: 表格数据管理，包括分页、排序、筛选
  - `useTableColumns`: 表格列配置管理
  - `useTableHeight`: 表格高度自适应
- **使用建议**:
  - 组合使用这三个钩子，实现完整的表格功能
  - 根据业务需求配置表格的分页、排序和筛选逻辑
  - 利用 `useTableHeight` 确保表格在不同屏幕尺寸下的良好显示

#### 图表数据处理
- **推荐钩子**:
  - `useChart`: 图表数据处理和配置
- **使用建议**:
  - 在需要展示图表的页面中使用
  - 配置图表的数据源、样式和交互行为
  - 处理图表的响应式布局

### 2.3 布局与主题场景

#### 布局管理
- **推荐钩子**:
  - `useLayoutHeight`: 布局高度管理
  - `useHeaderBar`: 头部栏状态管理
- **使用建议**:
  - 在需要自适应布局的页面中使用
  - 结合 `useLayoutHeight` 管理页面元素的高度
  - 利用 `useHeaderBar` 控制头部栏的状态和行为

#### 主题管理
- **推荐钩子**:
  - `useTheme`: 主题切换和管理
- **使用建议**:
  - 在需要主题切换功能的页面中使用
  - 结合设置面板组件使用，实现主题的实时切换
  - 处理主题变更对页面元素的影响

### 2.4 应用状态场景

#### 应用模式管理
- **推荐钩子**:
  - `useAppMode`: 应用模式管理
- **使用建议**:
  - 在需要根据不同应用模式调整功能的场景中使用
  - 处理不同模式下的 UI 和行为差异

#### 通用功能
- **推荐钩子**:
  - `useCommon`: 通用功能和工具方法
- **使用建议**:
  - 在需要使用通用功能的组件中使用
  - 利用其提供的工具方法简化业务逻辑

### 2.5 交互体验场景

#### 快速进入
- **推荐钩子**:
  - `useFastEnter`: 快速进入功能管理
- **使用建议**:
  - 在需要快速导航或快捷操作的场景中使用
  - 配置快速进入的目标和行为

#### 仪式效果
- **推荐钩子**:
  - `useCeremony`: 仪式效果管理
- **使用建议**:
  - 在需要特殊视觉效果的场景中使用
  - 配置仪式效果的触发条件和表现形式

## 3. 钩子函数使用最佳实践

### 3.1 钩子函数配置
- **合理配置参数**: 根据业务需求设置钩子函数的参数
- **利用返回值**: 正确使用钩子函数返回的状态和方法
- **遵循钩子 API**: 按照钩子函数的设计使用其功能

### 3.2 性能优化
- **按需使用**: 只在需要的组件中使用钩子函数
- **避免重复调用**: 合理组织代码，避免在同一组件中重复调用相同的钩子
- **优化依赖项**: 对于带有依赖项的钩子，确保依赖项的合理设置

### 3.3 代码组织
- **钩子组合**: 合理组合多个钩子函数，实现复杂功能
- **逻辑分离**: 将业务逻辑与 UI 逻辑分离，利用钩子函数管理业务逻辑
- **文档注释**: 为钩子函数的使用添加适当的注释

### 3.4 错误处理
- **异常捕获**: 对钩子函数可能抛出的异常进行适当处理
- **边界情况**: 考虑钩子函数在边界情况下的行为
- **错误反馈**: 提供清晰的错误反馈机制

## 4. 常见业务场景示例

### 4.3 内容管理系统
- **认证管理**: `useAuth` 用于权限控制
- **表格处理**: `useTable` 用于内容列表管理

## 5. 钩子函数使用检查清单

### 5.1 选择钩子前
- [ ] 明确业务需求的核心功能
- [ ] 评估现有钩子是否满足需求
- [ ] 考虑钩子的可维护性和扩展性

### 5.2 使用钩子时
- [ ] 正确配置钩子参数
- [ ] 合理使用钩子返回的状态和方法
- [ ] 确保钩子与其他钩子的兼容性

### 5.3 使用钩子后
- [ ] 测试钩子在不同场景下的表现
- [ ] 检查钩子的性能影响
- [ ] 文档化钩子的使用方式

## 6. 钩子函数使用禁忌

### 6.1 避免过度使用
- 不要在简单场景中使用复杂钩子
- 避免同时使用多个功能相似的钩子

### 6.2 避免不当定制
- 不要修改核心钩子的内部实现
- 避免通过覆盖钩子返回值的方式修改其行为

### 6.3 避免性能问题
- 不要在大型列表渲染中频繁调用钩子函数
- 避免在钩子函数中执行耗时操作

## 7. 总结

合理使用钩子函数是提高开发效率和保证代码质量的关键。通过遵循本规则，开发者可以：

1. **快速选择合适的钩子**: 根据业务场景选择最适合的钩子函数
2. **提高代码复用性**: 利用钩子函数封装和复用业务逻辑
3. **保证代码质量**: 遵循钩子函数使用的最佳实践
4. **提升开发效率**: 减少重复代码，专注于业务逻辑的实现

本规则将随着项目的发展和钩子函数库的更新而不断完善。
