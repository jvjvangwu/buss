# 业务组件使用规则

## 1. 组件选择原则

### 1.1 业务场景匹配
- **根据功能需求选择组件**: 明确业务需求的核心功能，选择最适合的组件
- **考虑用户体验**: 选择能够提供良好用户体验的组件
- **遵循项目规范**: 使用符合项目代码风格和命名规范的组件

### 1.2 组件优先级
1. **核心组件**: 优先使用 `src/components/core/` 下的组件
2. **业务组件**: 其次考虑 `src/components/business/` 下的组件
3. **自定义组件**: 最后考虑根据业务需求定制组件

## 2. 业务场景与组件匹配

### 2.1 数据展示场景

#### 仪表盘/数据概览
- **推荐组件**:
  - `ArtStatsCard`: 显示关键指标
  - `ArtBarChartCard`: 展示数据对比
  - `ArtLineChartCard`: 展示趋势数据
  - `ArtDonutChartCard`: 展示占比数据
- **使用建议**:
  - 组合使用多种卡片组件，形成完整的仪表盘
  - 根据数据类型选择合适的图表卡片

#### 数据列表
- **推荐组件**:
  - `ArtTable`: 展示结构化数据
  - `ArtTableHeader`: 提供搜索和筛选功能
- **使用建议**:
  - 对于大量数据，考虑分页和虚拟滚动
  - 为表格添加适当的排序和筛选功能

### 2.2 表单交互场景

#### 数据录入
- **推荐组件**:
  - `ArtForm`: 表单容器
  - `ArtSearchBar`: 搜索功能
- **使用建议**:
  - 合理组织表单字段，使用适当的表单验证
  - 对于复杂表单，考虑分步表单

#### 文件处理
- **推荐组件**:
  - `ArtExcelImport`: 批量导入数据
  - `ArtExcelExport`: 导出数据
  - `ArtCutterImg`: 图片上传和裁剪
- **使用建议**:
  - 提供清晰的文件格式说明
  - 处理文件上传的错误和边界情况

#### 内容编辑
- **推荐组件**:
  - `ArtWangEditor`: 富文本编辑
- **使用建议**:
  - 根据编辑需求配置编辑器功能
  - 实现内容的实时预览

### 2.3 导航布局场景

#### 页面导航
- **推荐组件**:
  - `ArtBreadcrumb`: 面包屑导航
  - `ArtHeaderBar`: 页面头部
  - `ArtSidebarMenu`: 侧边栏菜单
  - `ArtHorizontalMenu`: 水平菜单
- **使用建议**:
  - 根据应用规模选择合适的导航方式
  - 确保导航层次清晰，易于理解

#### 页面布局
- **推荐组件**:
  - `ArtPageContent`: 页面内容容器
  - `ArtWorkTab`: 标签页管理
- **使用建议**:
  - 保持页面布局的一致性
  - 合理使用标签页管理多任务

### 2.4 媒体处理场景

#### 图片处理
- **推荐组件**:
  - `ArtCutterImg`: 图片裁剪
  - `ArtImageCard`: 图片展示
- **使用建议**:
  - 优化图片加载性能
  - 提供适当的图片预览功能

#### 视频播放
- **推荐组件**:
  - `ArtVideoPlayer`: 视频播放器
- **使用建议**:
  - 确保视频格式兼容性
  - 提供基本的播放控制功能

### 2.5 反馈交互场景

#### 操作结果
- **推荐组件**:
  - `ArtResultPage`: 操作结果页面
- **使用建议**:
  - 清晰展示操作结果
  - 提供适当的后续操作选项

#### 异常处理
- **推荐组件**:
  - `ArtException`: 异常页面
- **使用建议**:
  - 提供友好的错误提示
  - 引导用户回到正常流程

### 2.6 其他场景

#### 回到顶部
- **推荐组件**:
  - `ArtBackToTop`: 回到顶部按钮
- **使用建议**:
  - 在长页面中使用，提升用户体验

#### 水印
- **推荐组件**:
  - `ArtWatermark`: 页面水印
- **使用建议**:
  - 在需要版权保护的页面使用
  - 确保水印不影响内容可读性

## 3. 组件使用最佳实践

### 3.1 组件配置
- **合理配置 props**: 根据业务需求设置组件属性
- **利用事件回调**: 正确处理组件事件
- **遵循组件 API**: 按照组件设计使用其功能

### 3.2 性能优化
- **按需加载**: 对于大型组件，考虑懒加载
- **避免过度渲染**: 合理使用 v-if 和 v-show
- **优化数据传递**: 避免不必要的 prop 传递

### 3.3 代码组织
- **组件封装**: 对于复杂业务逻辑，封装为业务组件
- **代码复用**: 提取通用逻辑为组合式函数
- **文档注释**: 为组件使用添加适当的注释

### 3.4 样式管理
- **优先使用 Tailwind CSS**: 利用工具类快速构建界面
- **合理使用 SCSS**: 对于复杂样式使用 SCSS
- **样式隔离**: 组件样式使用 scoped

## 5. 组件使用检查清单

### 5.1 选择组件前
- [ ] 明确业务需求的核心功能
- [ ] 评估现有组件是否满足需求
- [ ] 考虑组件的可维护性和扩展性

### 5.2 使用组件时
- [ ] 正确配置组件属性
- [ ] 处理组件事件和回调
- [ ] 确保组件与其他组件的兼容性

### 5.3 使用组件后
- [ ] 测试组件在不同场景下的表现
- [ ] 检查组件的性能影响
- [ ] 文档化组件的使用方式

## 6. 组件使用禁忌

### 6.1 避免过度使用
- 不要在简单场景中使用复杂组件
- 避免同时使用多个功能相似的组件

### 6.2 避免不当定制
- 不要修改核心组件的内部实现
- 避免通过 CSS 覆盖组件样式

### 6.3 避免性能问题
- 不要在大型列表中使用重型组件
- 避免不必要的组件嵌套

## 7. 总结

合理使用组件是提高开发效率和保证代码质量的关键。通过遵循本规则，开发者可以：

1. **快速选择合适的组件**：根据业务场景选择最适合的组件
2. **提高开发效率**：利用现有组件快速构建功能
3. **保证代码质量**：遵循组件使用的最佳实践
4. **提升用户体验**：使用设计良好的组件提供优质的用户界面

本规则将随着项目的发展和组件库的更新而不断完善。
