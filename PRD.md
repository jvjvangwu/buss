## 1. 项目概述

### 1.1 产品背景

建设一个企业级论坛系统，为用户提供信息交流、知识分享和互动讨论的平台。第一阶段聚焦于核心功能：用户管理和新闻发布，为后续社区功能奠定基础。

### 1.2 产品目标

| 目标类型 | 具体目标 | 衡量指标 |
|---------|---------|---------|
| 业务目标 | 搭建稳定可用的论坛基础平台 | 系统上线运行 |
| 用户目标 | 提供便捷的用户管理和新闻阅读体验 | 用户注册率 > 60% |
| 技术目标 | 构建可扩展的技术架构 | 支持后续功能迭代 |

### 1.3 目标用户

| 用户角色 | 描述 | 核心需求 |
|---------|------|---------|
| 管理员 | 系统管理者，负责内容审核和用户管理 | 用户管理、权限控制、内容管理 |
| 普通用户 | 论坛使用者，浏览和发布内容 | 注册登录、浏览新闻、个人信息管理 |

### 1.4 项目范围

**包含功能**:
- 用户注册/登录（账号密码 + 微信 OAuth）
- 角色与权限管理
- 用户信息维护
- 新闻发布与管理
- 新闻分类/标签管理
- 操作日志
- 系统配置
- 文件上传

**不包含功能**（后续阶段）:
- 帖子讨论功能
- 评论互动
- 私信系统
- 积分体系
- 通知推送

## 3. 用户故事

### 3.1 用户管理模块

#### UM-001 账号密码注册

**作为** 新用户  
**我想要** 通过用户名、密码、邮箱注册账号  
**以便于** 成为论坛会员并使用论坛功能

**验收条件**:
- 用户名：4-20 字符，支持字母、数字、下划线，唯一性校验
- 密码：8-32 字符，必须包含字母和数字，支持特殊字符
- 邮箱：标准邮箱格式，唯一性校验，需验证邮箱有效性
- 注册成功后自动登录，跳转到首页
- 注册失败时显示明确的错误提示

#### UM-002 账号密码登录

**作为** 已注册用户  
**我想要** 使用账号密码登录系统  
**以便于** 访问需要登录的功能

**验收条件**:
- 支持用户名或邮箱登录
- 密码错误时提示"用户名或密码错误"（不暴露具体错误）
- 连续 5 次密码错误，账号锁定 15 分钟
- 登录成功后跳转到来源页面或首页
- 支持"记住我"功能，延长 Token 有效期

#### UM-003 微信 OAuth 登录

**作为** 微信用户  
**我想要** 通过微信扫码授权登录  
**以便于** 快速登录无需记忆密码

**验收条件**:
- 显示微信二维码，用户扫码授权
- 新用户首次微信登录自动创建账号
- 已绑定微信的用户直接登录成功
- 二维码有效期 5 分钟，过期需刷新
- 支持微信账号与已有账号绑定

#### UM-005 角色管理

**作为** 系统管理员  
**我想要** 创建和管理角色  
**以便于** 为用户分配不同的权限

**验收条件**:
- 角色名称：2-50 字符，唯一性校验
- 角色标识：用于代码中权限判断，如 `ROLE_ADMIN`
- 角色描述：可选，最多 200 字符
- 支持启用/禁用角色
- 禁用角色后，该角色用户无法登录

#### UM-006 权限分配

**作为** 系统管理员  
**我想要** 为角色分配菜单和按钮权限  
**以便于** 控制用户可访问的功能

**验收条件**:
- 权限树形结构展示，支持全选/反选
- 菜单权限：控制用户可见的菜单项
- 按钮权限：控制页面内按钮的显示和操作权限
- 权限变更后，用户下次登录或刷新页面生效
- 系统内置角色（超级管理员）不可编辑权限

#### UM-007 用户列表

**作为** 系统管理员  
**我想要** 查看和管理用户列表  
**以便于** 了解用户情况并进行管理

**验收条件**:
- 列表展示：用户名、昵称、邮箱、角色、状态、注册时间
- 支持按用户名、邮箱搜索
- 支持按角色、状态筛选
- 支持分页，每页 10/20/50/100 条可选
- 支持导出用户列表（Excel 格式）

#### UM-010 个人信息修改

**作为** 登录用户  
**我想要** 修改我的个人信息  
**以便于** 完善个人资料

**验收条件**:
- 可修改：昵称、头像、个人简介、性别
- 昵称：2-20 字符，可重复
- 头像：上传图片，支持 JPG/PNG，最大 2MB
- 个人简介：最多 200 字符
- 修改成功后即时生效

### 3.2 新闻发布模块

#### NP-001 新闻创建

**作为** 内容管理员  
**我想要** 使用富文本编辑器创建新闻  
**以便于** 发布新闻内容供用户阅读

**验收条件**:
- 新闻标题：5-100 字符，必填
- 新闻摘要：最多 200 字符，可选，未填写自动截取正文前 200 字
- 新闻内容：富文本，支持图片、链接、格式化，必填
- 新闻封面：上传图片，支持 JPG/PNG，最大 2MB，可选
- 新闻分类：单选，必填
- 新闻标签：多选，最多 5 个
- 发布状态：草稿/已发布
- 保存草稿不发布，仅管理员可见

#### NP-006 新闻列表（用户端）

**作为** 论坛用户  
**我想要** 浏览新闻列表  
**以便于** 查看最新和感兴趣的新闻

**验收条件**:
- 列表展示：封面图、标题、摘要、分类、发布时间、阅读量
- 支持按分类筛选
- 支持按时间排序（最新/热门）
- 置顶新闻优先显示，带"置顶"标识
- 分页加载，每页 10 条
- 点击新闻卡片跳转到详情页

#### NP-007 新闻详情

**作为** 论坛用户  
**我想要** 查看新闻详细内容  
**以便于** 阅读完整新闻

**验收条件**:
- 展示：标题、作者、发布时间、阅读量、分类、标签、正文内容
- 正文支持富文本渲染（图片、链接、格式化）
- 阅读量自动 +1
- 底部显示相关新闻推荐（同分类或同标签）
- 支持分享到微信（生成二维码）

#### NP-008 分类管理

**作为** 内容管理员  
**我想要** 管理新闻分类  
**以便于** 组织新闻内容

**验收条件**:
- 分类名称：2-20 字符，唯一性校验
- 分类排序：数字，用于列表显示顺序
- 分类状态：启用/禁用
- 禁用分类后，该分类新闻不在用户端显示
- 删除分类时，需选择将分类下新闻迁移到其他分类或一并删除

#### NP-009 标签管理

**作为** 内容管理员  
**我想要** 管理新闻标签  
**以便于** 为新闻打标签便于检索

**验收条件**:
- 标签名称：2-20 字符，唯一性校验
- 标签颜色：预设颜色选择，用于前端展示
- 标签状态：启用/禁用
- 删除标签时，解除标签与新闻的关联关系

### 3.3 系统功能模块

#### SF-001 操作日志记录

**作为** 系统管理员  
**我想要** 系统自动记录用户操作行为  
**以便于** 追踪问题和审计

**验收条件**:
- 记录内容：操作人、操作时间、操作类型、操作对象、操作结果、IP 地址
- 操作类型：登录、登出、新增、修改、删除、查看等
- 关键操作必须记录：用户登录/登出、用户管理、角色权限变更、新闻发布/删除
- 日志保留 90 天，自动清理过期日志
- 日志写入不影响业务性能（异步写入）

#### SF-004 文件上传

**作为** 系统用户  
**我想要** 上传图片和文档  
**以便于** 在新闻内容或个人资料中使用

**验收条件**:
- 支持格式：JPG、PNG、GIF、PDF、DOC、DOCX、XLS、XLSX
- 单文件大小限制：图片 2MB，文档 10MB
- 返回文件访问 URL
- 图片自动压缩（宽度 > 1920px 自动缩放）
- 文件命名：时间戳 + 随机字符串，避免冲突
- 上传目录按日期分文件夹存储

## 5. 非功能性需求

### 5.1 性能需求

| 指标 | 目标值 | 说明 |
|-----|-------|-----|
| 页面加载时间 | < 2 秒 | 首屏加载完成 |
| API 响应时间 | < 500ms | 95% 请求响应时间 |
| 并发用户数 | 500 | 系统稳定支持的同时在线用户 |
| 数据库查询 | < 100ms | 单次查询响应时间 |
| 文件上传 | < 5 秒 | 2MB 图片上传完成 |

### 5.2 安全需求

| 安全项 | 要求 |
|-------|-----|
| 密码存储 | BCrypt 加密，加盐哈希 |
| 传输加密 | HTTPS 全站加密 |
| 身份认证 | JWT Token，有效期 2 小时，刷新 Token 7 天 |
| XSS 防护 | 富文本内容过滤，转义危险标签 |
| CSRF 防护 | Token 校验机制 |
| SQL 注入防护 | 使用参数化查询，禁止拼接 SQL |
| 文件上传安全 | 文件类型校验，文件内容检测，禁止执行权限 |
| 敏感信息脱敏 | 日志中密码、Token 等敏感信息脱敏 |
| 接口限流 | 登录接口限流，防止暴力破解 |

### 5.3 可用性需求

| 指标 | 目标值 |
|-----|-------|
| 系统可用性 | 99.5% |
| 故障恢复时间 | < 30 分钟 |
| 数据备份 | 每日自动备份，保留 7 天 |
| 错误处理 | 友好错误提示，不暴露系统信息 |

### 5.4 可维护性需求

| 要求 | 说明 |
|-----|-----|
| 代码规范 | 遵循阿里巴巴 Java 开发手册、Vue 官方风格指南 |
| 代码注释 | 关键逻辑必须有注释，复杂算法详细说明 |
| 日志规范 | 统一日志格式，区分日志级别 |
| 文档要求 | API 文档（Swagger）、部署文档、运维手册 |
| 版本控制 | Git 分支管理规范，提交信息规范 |

### 5.5 可扩展性需求

| 要求 | 说明 |
|-----|-----|
| 架构设计 | 前后端分离，模块化设计 |
| 数据库设计 | 预留扩展字段，支持分表分库 |
| 接口设计 | RESTful 风格，版本控制 |

## 6. 技术架构

### 6.1 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                      前端层                              │
│  Vue3 + TypeScript + Vite + Element Plus + Pinia        │
└─────────────────────────────────────────────────────────┘
                            │
                            │ HTTP/HTTPS
                            ▼
┌─────────────────────────────────────────────────────────┐
│                      网关层                              │
│                    Nginx                                │
└─────────────────────────────────────────────────────────┘
                            │
                            │ 反向代理
                            ▼
┌─────────────────────────────────────────────────────────┐
│                      后端层                              │
│  Spring Boot 3 + Spring Security + JWT + MyBatis-Plus   │
└─────────────────────────────────────────────────────────┘
                            │
                            │ JDBC
                            ▼
┌─────────────────────────────────────────────────────────┐
│                      数据层                              │
│              PostgreSQL 15+ + Redis（缓存）              │
└─────────────────────────────────────────────────────────┘
```

## 8. API 设计概要

### 8.1 API 规范

- 基础路径: `/api/v1`
- 认证方式: Bearer Token (JWT)
- 请求格式: JSON
- 响应格式: JSON
- 响应结构:
```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "timestamp": 1703275200000
}
```
