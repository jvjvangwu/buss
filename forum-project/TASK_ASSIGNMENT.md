## 每日工作流程
1. **早上9:30**：每日站会（15分钟）
   - 昨天完成了什么
   - 今天计划做什么
   - 遇到什么困难

2. **工作期间**：
   - 使用Git进行版本控制
   - 提交代码前进行自测
   - 遇到问题及时沟通

3. **代码提交**：
   - 每天至少提交一次
   - 提交信息规范：`feat: 描述` / `fix: 描述` / `docs: 描述`
   - 创建Pull Request进行代码评审

## 沟通渠道
- **即时沟通**：使用团队聊天工具
- **文档共享**：项目文档在Git仓库
- **会议记录**：每日站会记录在`daily-standup.md`
- **问题跟踪**：使用GitHub Issues
