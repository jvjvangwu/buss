# 论坛系统部署文档和运维手册

## 1. 项目概述

本项目是一个基于 Spring Boot 3.1.5 和 Vue 3.4 的前后端分离论坛系统，主要功能包括：

- 用户认证与授权（登录、注册、角色权限管理）
- 新闻管理（发布、编辑、删除、置顶、热门）
- 分类和标签管理
- 文件上传和管理
- 操作日志记录
- 个人中心（个人信息修改、密码修改、消息通知）

## 3. 后端部署步骤

### 3.1 环境配置

1. **安装 Java 17**
   - 下载并安装 JDK 17
   - 配置环境变量 `JAVA_HOME`

2. **安装 PostgreSQL 15**
   - 下载并安装 PostgreSQL 15
   - 创建数据库 `forum_db`
   - 创建用户 `forum_user` 并设置密码

3. **安装 Redis（可选）**
   - 下载并安装 Redis 6.0+
   - 启动 Redis 服务

### 3.2 项目构建

1. **克隆项目**
   ```bash
   git clone <项目地址>
   cd forum-backend
   ```

2. **配置数据库连接**
   - 编辑 `src/main/resources/application.yml` 文件
   - 修改数据库连接信息

3. **构建项目**
   ```bash
   mvn clean package -DskipTests
   ```

### 3.3 部署运行

1. **运行 jar 包**
   ```bash
   java -jar target/forum-backend-1.0.0.jar
   ```

2. **使用 systemd 管理服务（Linux）**
   - 创建服务文件 `/etc/systemd/system/forum-backend.service`
   - 启动服务
   ```bash
   systemctl start forum-backend
   systemctl enable forum-backend
   ```

## 4. 前端部署步骤

### 4.1 环境配置

1. **安装 Node.js 18**
   - 下载并安装 Node.js 18+
   - 验证安装
   ```bash
   node -v
   npm -v
   ```

### 4.2 项目构建

1. **克隆项目**
   ```bash
   git clone <项目地址>
   cd forum-frontend
   ```

2. **安装依赖**
   ```bash
   npm install
   ```

3. **配置 API 地址**
   - 编辑 `.env.production` 文件
   - 修改 `VITE_API_BASE_URL` 为后端 API 地址

4. **构建项目**
   ```bash
   npm run build
   ```

### 4.3 部署运行

1. **部署到 Nginx**
   - 配置 Nginx 虚拟主机
   - 将 `dist` 目录部署到 Nginx 根目录

2. **Nginx 配置示例**
   ```nginx
   server {
       listen 80;
       server_name forum.example.com;
       
       location / {
           root /path/to/forum-frontend/dist;
           index index.html;
           try_files $uri $uri/ /index.html;
       }
       
       location /api {
           proxy_pass http://localhost:8080/api;
           proxy_set_header Host $host;
           proxy_set_header X-Real-IP $remote_addr;
           proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
       }
   }
   ```

## 5. 数据库配置

### 5.1 数据库初始化

1. **创建数据库**
   ```sql
   CREATE DATABASE forum_db;
   ```

2. **创建用户**
   ```sql
   CREATE USER forum_user WITH PASSWORD 'your_password';
   GRANT ALL PRIVILEGES ON DATABASE forum_db TO forum_user;
   ```

3. **启动应用**
   - 应用启动时会自动创建表结构

### 5.2 数据迁移

- 使用 MyBatis-Plus 的自动迁移功能
- 或使用数据库迁移工具（如 Flyway）

## 6. 常见问题及解决方案

### 6.1 后端问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 数据库连接失败 | 连接配置错误 | 检查 application.yml 中的数据库连接配置 |
| 端口被占用 | 8080 端口已被其他服务占用 | 修改 application.yml 中的 server.port 配置 |
| 权限不足 | 用户权限配置错误 | 检查角色权限配置 |

### 6.2 前端问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| API 调用失败 | API 地址配置错误 | 检查 .env.production 中的 API 地址 |
| 页面白屏 | 构建失败或资源加载错误 | 检查构建日志和浏览器控制台 |
| 样式异常 | CSS 加载失败 | 检查静态资源路径配置 |

## 7. 运维指南

### 7.1 日志管理

- 后端日志：`logs/forum-backend.log`
- 前端日志：浏览器控制台
- 操作日志：通过管理后台查看

1. **后端优化**
   - 使用 Redis 缓存热点数据

2. **前端优化**

### 7.3 安全措施

1. **后端安全**
   - 使用 HTTPS
   - 实施 CORS 策略
   - 防止 SQL 注入
   - 防止 XSS 攻击

2. **前端安全**
   - 输入验证
   - 防止 CSRF 攻击
   - 敏感信息保护

### 7.4 监控与告警

- 使用 Prometheus + Grafana 监控系统
- 设置关键指标告警
- 定期备份数据库

### 10.3 部署脚本

#### 后端部署脚本

```bash
#!/bin/bash

# 停止服务
systemctl stop forum-backend

# 备份旧版本
if [ -f "target/forum-backend-1.0.0.jar" ]; then
    cp target/forum-backend-1.0.0.jar target/forum-backend-1.0.0.jar.bak
fi

# 构建项目
mvn clean package -DskipTests

# 启动服务
systemctl start forum-backend

# 查看状态
systemctl status forum-backend
```

#### 前端部署脚本

```bash
#!/bin/bash

# 安装依赖
npm install

# 构建项目
npm run build

# 复制到 Nginx 目录
cp -r dist/* /path/to/nginx/html/

# 重启 Nginx
systemctl restart nginx
```
